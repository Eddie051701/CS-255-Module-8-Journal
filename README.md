# CS-255-Module-8-Journal

Project Summary

DriverPass is a firm providing training to assist students in preparing for DMV driving tests. It required a cloud-based web application to enable customers to sign up for training packages, book driving sessions, complete online practice exams, and monitor their status. Admin features were added to manage bookings, generate reports, update DMV content, and accommodate other user roles (customers, secretaries, the owner, and the IT officer).

What I Did Well

I think I did a good job of collecting and arranging the client's specifications. I reviewed the interview transcript thoroughly and converted the client's needs into distinct functional and non-functional specifications. I made UML diagrams which illustrated how users would be able to interact with the system, and how the various components of the system worked together to satisfy the needs of the client.

Improvements and Revisions

If I had another opportunity to revise something from my original work, I would add extra detail and relationships between the use cases and classes on my UML diagram. My diagrams were able to capture the majority of the overall system functions, but extra details can make them easier to read and provide a much more accurate view of what the system does.

Understanding User Requirements

I have determined user requirements by analyzing the Driver Pass interview transcript to identify the goals, troubles, and desires of the client. I then converted these requirements into functions of the system. Understanding user requirements is very important since a system can only succeed if it resolves the consumer's problem and gives the users a good experience each time they use it.

Design Software Methodology

My methodology of software design starts with understanding the business requirement of the client and determining the needs of all parties. Once I have finished creating a model and diagrams illustrating how users will interact with the system and how the components will work together to fulfill the needs of clients. Then I proceed to implement. For future projects I will continue to utilize similar methods like requirements analysis, use cases modeling, activity diagrams, sequence diagrams, and class diagrams so I can be sure that the systems are structured properly, and can grow and comply with the expectation of users. Proper planning and documentation helps reduce issues during development and allows us to develop systems that are simpler to maintain and expand as time progresses.
